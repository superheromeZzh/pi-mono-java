ADR-0004 · custom-provider 模型置顶排序

Status: Accepted · Date: 2026-06-01 · 设计文档: list-models-usable-filter

Context

用户希望自己加的 custom 模型在列表里更显眼。原排序是 provider.value() 然后 id， custom 会按字母散落在中间。

Decision

引入 CUSTOM_FIRST 比较器：Provider.CUSTOM 优先（rank 0），其余按 provider.value() 再按 id。getAvailableModels() 与 getAllModels() 都应用，保证默认列表与 all:true 列表排序一致。

考虑过的选项

A · 比较器置顶，两个方法都应用（选中）
服务端统一保证次序，前端无需自行排序
默认列表与 all:true 列表次序一致
多了一个静态比较器常量
B · 让前端排序
服务端零改动
每个前端各实现一遍，WS 与 REST 客户端易不一致

Rationale

排序是展示契约的一部分，放在唯一的 service 出口最稳。两个列表方法共用同一比较器避免次序漂移。

Consequences

＋ custom 永远在最前，跨 WS / REST 一致。
－ 依赖旧「严格 provider 字母序」的前端需适配（custom 不再按字母排）。
