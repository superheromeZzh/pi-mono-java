ADR-0002 · 凭证过滤烘焙进共享 getAvailableModels（影响 WS + REST）

Status: Accepted · Date: 2026-06-01 · 设计文档: list-models-usable-filter

Context

ModelCatalogService.getAvailableModels() 被两个调用方共用：WS handleListModels（all:false 分支）与 REST SettingsHandler.buildModelsSnapshot。两者都希望默认只展示可用模型。

Decision

把凭证过滤直接写进 getAvailableModels()：候选集（白名单展开 + custom）→ 按 hasCredentials 过滤 → custom 置顶排序。getAllModels() 保持不过滤，作为 list_models all:true 的逃生口。

考虑过的选项

A · 改共享 getAvailableModels（选中）
一处修改，WS 与 REST 行为天然一致
任何未来新调用方都会继承凭证过滤（目前无此类调用方）
B · 新增 getUsableModels()，仅 WS 调用
保留 getAvailableModels 纯白名单语义
WS 与 REST 分叉、两套 wire 逻辑、getAvailableModels 名实不符

Rationale

两个现有调用方诉求相同，集中到共享 service 是「单一事实源」的延续。顺带订正 ModelCatalogService 类 Javadoc 中「picker / -m flag / WS 三处共用同一过滤」的失实表述—— picker 与 -m 实际直查 ModelRegistry，不在本 service 范围。

Consequences

＋ WS 与 REST 永远展示同一集合。
－ getAvailableModels 现在隐含凭证过滤，调用方若需全量必须显式走 getAllModels()。
