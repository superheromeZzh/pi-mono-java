ADR-0003 · enabledModels 白名单保留为交集（凭证 ∩ 白名单）

Status: Accepted · Date: 2026-06-01 · 设计文档: list-models-usable-filter

Context

需求是「返回可用模型，而不是白名单模型」。「有凭证」与「在 enabledModels 白名单内」是两个正交维度—— 需要决定引入凭证过滤后，既有的 enabledModels curation 功能是否保留。

Decision

保留 enabledModels 为**交集**：配了白名单时，默认列表 = 白名单 ∩ 有凭证；没配白名单（常见）时 = 所有有凭证的模型。custom 始终包含。

考虑过的选项

A · 凭证 ∩ 白名单（选中）
非破坏性，保留既有 curation 功能
任何情况下都满足「只返回可用模型」（交集是可用集的子集）
配过白名单的用户，有凭证但不在白名单的模型仍不显示
B · 完全忽略 enabledModels，纯凭证过滤
最贴近字面「不要白名单」
配过白名单做 curation 的用户会发现筛选静默失效

Rationale

用户在 Q1 重述时把「可用」纯粹用凭证来源定义，未提白名单。交集方案既兑现「只返回可用」，又不悄悄废弃一个用户主动配置过的特性，代价仅是「有凭证但被 curation 排除」的少数模型不显示——这与 curation 的本意一致。

Consequences

＋ 既有 enabledModels 用户行为平滑。
－ 若后续确认要「完全忽略白名单」，改动仅几行（删去 narrowToEnabled 调用），届时本 ADR 置 Superseded 并新建 ADR 记录。
