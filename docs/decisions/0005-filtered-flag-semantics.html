ADR-0005 · filtered 标志维持白名单语义，不新增字段

Status: Accepted · Date: 2026-06-01 · 设计文档: list-models-usable-filter

Context

响应里的 filtered 原义是「enabledModels 是否生效」。引入凭证过滤后，默认列表总是被凭证收窄—— 需要决定 filtered 是否应反映凭证过滤，或是否新增字段。

Decision

filtered 维持白名单语义：仅表示 enabledModels 是否生效。凭证过滤是默认列表的恒定行为， 通过契约文档说明，不新增响应字段。要看全量走 all:true。

考虑过的选项

A · 维持白名单语义 + 文档说明（选中）
不破坏既有 filtered 消费方
filtered:false 时列表其实仍被凭证收窄，语义需文档澄清
B · 新增 credentialFiltered 字段
显式暴露凭证过滤状态
契约面变大、两个 schema 都要改，多数前端用不到
C · 扩大 filtered 含义为「任意收窄」
单字段表达「非全量」
悄悄改变既有字段语义，是破坏性变更

Rationale

默认列表「按凭证过滤」是稳定不变的前置条件，不是一个会翻转的状态位，没必要单列字段；而改 filtered 既有含义会破坏兼容。文档澄清成本最低、契约最稳。

Consequences

＋ 现有 filtered 客户端无需改动。
－ 前端无法仅凭响应区分「因凭证被过滤」与「全量」，须知晓默认即凭证过滤这一约定。
